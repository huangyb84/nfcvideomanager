# 🔗 NFC 与视频映射管理模块

## 📋 功能概述

本模块实现了 **NFC 链接** 与 **视频链接** 之间的一对一映射关系管理，确保：

- ✅ 每个 NFC 链接准确映射到一个视频链接
- ✅ 映射关系持久化存储（数据库）
- ✅ 支持查询、添加、修改、删除操作
- ✅ 系统重启后映射关系不丢失

---

## 🎯 核心功能

### 1. 查询映射关系
- 查看所有已建立的映射
- 搜索特定映射
- 统计映射数量

### 2. 添加映射
- 选择未绑定的 NFC 链接
- 选择要绑定的视频链接
- 建立一对一映射关系

### 3. 修改映射
- 保持 NFC 链接不变
- 更换绑定的视频链接
- 自动更新映射关系

### 4. 删除映射
- 解除 NFC 与视频的绑定
- NFC 链接恢复为未绑定状态
- 视频链接不受影响

## 📖 操作步骤

### 场景 1：建立新的映射关系

**前提条件：**
- 已批量生成 NFC 链接（未绑定状态）
- 已上传视频生成视频链接

**操作步骤：**

1. 访问 `mapping-manager.html`
2. 点击 **➕ 添加映射**
3. 选择 NFC 链接（下拉框）
   - 只能选择未绑定的 NFC 链接
4. 选择视频链接（下拉框）
   - 显示所有已上传的视频
5. 点击 **添加映射**

**结果：**
```
✅ 映射关系添加成功！
NFC 链接：/v/abc123 → 视频链接：产品介绍.mp4
```

---

### 场景 2：更换视频（修改映射）

**操作步骤：**

1. 找到要修改的映射
2. 点击 **✏️ 编辑**
3. 选择新的视频链接
4. 点击 **更新映射**

**效果：**
```
✅ 映射关系已更新！
NFC 链接：/v/abc123（保持不变）
新视频：企业宣传.mp4（已更换）
```

---

### 场景 3：解除映射关系

**操作步骤：**

1. 找到要删除的映射
2. 点击 **🗑️ 删除**
3. 确认删除

**结果：**
```
✅ 映射关系已删除
NFC 链接：/v/abc123 → 未绑定状态
视频链接：产品介绍.mp4 → 未被绑定
```

---

### 场景 4：搜索映射

**操作步骤：**

1. 在搜索框输入关键词
2. 实时过滤显示结果

**支持的搜索条件：**
- NFC 链接标题
- NFC 短 ID（如 abc123）

## 🗄️ 数据存储

### 数据库表结构

**nfc_links 表**（NFC 链接）
```sql
CREATE TABLE nfc_links (
  id BIGSERIAL PRIMARY KEY,
  short_id TEXT UNIQUE NOT NULL,      -- 短 ID
  nfc_url TEXT NOT NULL,               -- 完整链接
  title TEXT,                          -- 标题
  is_active BOOLEAN DEFAULT true,      -- 是否启用
  created_at TIMESTAMP DEFAULT NOW()   -- 创建时间
);
```

**video_contents 表**（视频内容）
```sql
CREATE TABLE video_contents (
  id BIGSERIAL PRIMARY KEY,
  nfc_link_id BIGINT REFERENCES nfc_links(id),  -- 🔗 外键关联
  video_url TEXT NOT NULL,                       -- 视频链接
  title TEXT,                                    -- 视频标题
  is_current BOOLEAN DEFAULT true,               -- 是否当前绑定
  created_at TIMESTAMP DEFAULT NOW()             -- 绑定时间
);
```

### 映射关系示意

```
nfc_links 表                  video_contents 表
┌─────────────────┐          ┌─────────────────────┐
│ id: 1           │◄────────┤ nfc_link_id: 1      │
│ short_id: abc123│  映射    │ video_url: https:// │
│ nfc_url: /v/abc │  关系    │ title: 产品介绍     │
│ title: 链接 1    │          │ is_current: true    │
└─────────────────┘          └─────────────────────┘
```

## 🔧 API 接口

### 1. 添加映射
```http
POST /api/nfc-links/:id/bind
Content-Type: application/json

{
  "videoUrl": "https://...",
  "title": "产品介绍"
}
```

**响应：**
```json
{
  "success": true,
  "data": {
    "nfcLink": { ... },
    "videoContent": { ... }
  }
}
```

---

### 2. 查询所有映射
```http
GET /api/nfc-links
```

**响应：**
```json
{
  "success": true,
  "data": [
    {
      "id": 1,
      "short_id": "abc123",
      "nfc_url": "/v/abc123",
      "current_video": {
        "video_url": "https://...",
        "title": "产品介绍"
      }
    }
  ]
}
```

---

### 3. 更新映射
```http
POST /api/nfc-links/:id/bind
Content-Type: application/json

{
  "videoUrl": "https://new-video.mp4",
  "title": "新视频"
}
```

**效果：** 自动将旧视频标记为非当前，创建新映射

---

### 4. 删除映射
```http
DELETE /api/nfc-links/:id/unbind
```

**效果：** 将当前视频标记为非当前，解除绑定

## 📊 数据持久化

### 持久化机制

1. **数据库存储**
   - 映射关系存储在 Supabase 数据库
   - 使用外键关联保证数据完整性
   - 支持事务操作

2. **自动备份**
   - Supabase 自动备份数据
   - 支持时间点恢复

3. **系统重启**
   - 重启后数据不丢失
   - 重新启动时自动加载映射关系

---

## ✅ 数据完整性保证

### 1. 外键约束
```sql
nfc_link_id BIGINT REFERENCES nfc_links(id) ON DELETE CASCADE
```
- 删除 NFC 链接时，自动删除关联的视频内容
- 防止孤儿数据

### 2. 唯一性约束
```sql
short_id TEXT UNIQUE NOT NULL
```
- 每个 NFC 链接的短 ID 唯一
- 防止重复生成

### 3. 状态标记
```sql
is_current BOOLEAN DEFAULT true
```
- 只标记一个视频为当前绑定
- 支持历史视频记录

## 📈 统计信息

### 实时统计面板

| 指标 | 说明 | 计算方式 |
|------|------|----------|
| 总映射数 | 已建立的映射关系数量 | `COUNT(current_video IS NOT NULL)` |
| NFC 链接总数 | 所有 NFC 链接数量 | `COUNT(nfc_links)` |
| 视频链接总数 | 所有视频数量 | `COUNT(video_contents)` |
| 未映射 NFC | 未绑定的 NFC 链接数 | `COUNT(current_video IS NULL)` |

## 🔍 故障排查

### 问题 1：无法添加映射

**错误：** "NFC 链接不存在"

**检查：**
1. 确认 NFC 链接已生成
2. 确认 NFC 链接状态为启用
3. 查看数据库是否有数据

---

### 问题 2：映射后无法播放

**错误：** "未绑定视频"

**检查：**
1. 确认映射已成功建立
2. 查看 `current_video` 字段是否为空
3. 检查视频链接是否有效

---

### 问题 3：统计数据不准确

**解决：**
1. 刷新页面重新加载
2. 检查数据库连接
3. 查看 API 返回数据
